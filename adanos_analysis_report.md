# Анализ влияния AI Adanos на торговлю бота и рекомендации

## Введение
Данный отчет анализирует использование внешнего сентимент-сервиса Adanos в PRD-BOT, его влияние на торговые решения и проблемы, выявленные в ходе работы.

## Как Adanos используется в боте
*   **Модуль `prd_agent/signals/external_sentiment_agent.py`:** Этот модуль отвечает за запросы к API Adanos и обработку его ответов.
*   **Параметры из `config.yaml`:**
    *   `external_sentiment.adanos.enabled`: Включает/отключает модуль Adanos.
    *   `external_sentiment.adanos.api_key_env`: Имя переменной окружения для API-ключа Adanos.
    *   `external_sentiment.adanos.base_url`: Базовый URL для API Adanos.
    *   `external_sentiment.adanos.min_buzz_score`: Минимальный "buzz score" для учета сигнала (по умолчанию 55).
    *   `external_sentiment.adanos.min_sentiment_abs`: Минимальное абсолютное значение сентимента для учета сигнала (по умолчанию 0.15).
    *   `external_sentiment.adanos.trending_limit`: Количество трендовых символов, запрашиваемых у Adanos (по умолчанию 10).
    *   `external_sentiment.cache_sec`: Частота запросов к внешним сервисам (по умолчанию 900 секунд или 15 минут).
*   **Механизм генерации сигналов:**
    *   `_fetch_adanos_trending` запрашивает трендовые крипто-токены с Reddit.
    *   `_adanos_to_signal` преобразует эти трендовые данные в `UnifiedSignal`.
    *   Сигналы фильтруются по `min_buzz_score` и `min_sentiment_abs`.
    *   `source` этих сигналов устанавливается как `"adanos_reddit"`.
    *   `confidence`: Вычисляется на основе `0.55 + abs(sent_f) * 0.25 + buzz / 500.0`.
    *   **SL/TP:** Уровни Stop Loss и Take Profit устанавливаются на основе текущей цены и **фиксированного `atr_pct` (0.006)**, который не адаптируется к текущей волатильности рынка.

## Выявленные проблемы с AI Adanos
*   **Превышение лимита API (`HTTP 429`):** В логах бота регулярно фиксируются ошибки `Adanos trending HTTP 429: Monthly API limit exceeded`. Это означает, что бот **не получает сигналы от Adanos**, как только исчерпает месячный лимит запросов. Влияние Adanos на торговлю становится нулевым.
*   **Фиксированный расчет SL/TP:** Использование фиксированного `atr_pct` (0.006) для определения SL/TP является неоптимальным. Это может привести к слишком узким SL/TP на высоковолатильном рынке или слишком широким на низковолатильном.

## Степень влияния Adanos на торговлю
*   **В текущем состоянии влияние Adanos на торговлю практически нулевое или негативное.** Бот не получает сигналов большую часть времени из-за превышения лимитов. Когда сигналы все же поступают, их SL/TP могут быть неоптимальными.
*   **Ценность сигналов:** Без активных сигналов сложно оценить их фактическую ценность. Виртуальная оценка не проводилась специфично для `adanos_reddit`.

## Рекомендации по AI Adanos

1.  **Приоритет: Устранить проблему с лимитом API:**
    *   **Немедленное действие:** Увеличить `external_sentiment.cache_sec` с 900 до **3600** (1 час) в `config.yaml`. Это значительно снизит частоту запросов к Adanos API и должно помочь оставаться в рамках месячного лимита.
    *   **Дополнительное действие (по желанию):** Уменьшить `external_sentiment.adanos.trending_limit` (сейчас 10) до 5 или 3, чтобы снизить объем собираемых данных за один запрос, если это также влияет на лимиты.
    *   **Альтернатива:** Если сигналы от Adanos окажутся очень ценными, рассмотреть возможность приобретения платного тарифа для увеличения лимитов. Если нет - отключить Adanos (`external_sentiment.adanos.enabled: false`).

2.  **Провести анализ эффективности сигналов Adanos (после решения проблемы с лимитом):**
    *   **Рекомендуется:** После активации Adanos (когда не будет ошибок 429) собирать данные за 1-2 недели и затем провести анализ Win Rate и PnL сигналов с `source="adanos_reddit"`. Это позволит объективно оценить, насколько ценны эти сигналы.

3.  **Улучшить расчет SL/TP для сигналов Adanos (если Adanos будет оставлен):**
    *   **Предложение:** Модифицировать функцию `_adanos_to_signal` в `prd_agent/signals/external_sentiment_agent.py` для использования **динамического расчета SL/TP**. Например, вместо фиксированного `atr_pct` (0.006) использовать текущее значение ATR рынка для символа, чтобы SL/TP адаптировались к текущей волатильности.
    *   **Предложение:** Возможность настройки множителей ATR для SL/TP через `config.yaml` в секции `external_sentiment.adanos`.

4.  **Косвенное влияние:** Сигналы Adanos с `source="adanos_reddit"` проходят через `quality_gate` и `risk_pipeline` бота, так что их качество все-таки перепроверяется.

## Вывод
В текущем состоянии AI Adanos не является полноценным советником из-за проблем с лимитами. Для того чтобы понять его реальную ценность, необходимо сначала решить технические сложности с доступом к API и, возможно, улучшить логику установки SL/TP для его сигналов.
