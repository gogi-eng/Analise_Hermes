## 3. Анализ влияния AI Adanos

**Как Adanos используется:**
*   Adanos предоставляет сигналы на основе трендинга в Reddit Crypto, фильтруемые по `min_buzz_score` и `min_sentiment_abs`.
*   Сигналы от Adanos маркируются как `source="adanos_reddit"`.
*   SL/TP для сигналов Adanos устанавливаются на основе **фиксированного `atr_pct` (0.006)**.

**Проблемы с Adanos:**
*   **Превышение лимита API (`HTTP 429`):** Бот регулярно не получает сигналы от Adanos из-за превышения месячного лимита. Влияние Adanos на торговлю становится нулевым или минимальным.
*   **Неоптимальный расчет SL/TP:** Использование фиксированного `atr_pct` для определения SL/TP не адаптируется к текущей волатильности рынка.

**Степень влияния Adanos:**
*   **В текущем состоянии влияние Adanos на торговлю практически нулевое или негативное.** Бот не получает сигналов большую часть времени из-за превышения лимитов. Когда сигналы все же поступают, их SL/TP могут быть неоптимальными.

**Рекомендации:**
*   **Приоритет: Устранить проблему с лимитом API (HTTP 429):**
    *   **Немедленное действие (Конфигурация):** Увеличить `external_sentiment.cache_sec` в `config.yaml` с 900 до **3600** (1 час). Это значительно снизит частоту запросов.
    *   **Дополнительное действие (по желанию):** Уменьшить `external_sentiment.adanos.trending_limit` (сейчас 10) до 5 или 3.
    *   **Альтернатива:** Если сигналы Adanos не оправдают ожиданий, отключить его (`external_sentiment.adanos.enabled: false`).
*   **После решения проблемы с лимитом:** Провести анализ эффективности сигналов Adanos (Win Rate, PnL) за период его корректной работы.
*   **Улучшить расчет SL/TP для Adanos (если будет оставлен):** Модифицировать функцию `_adanos_to_signal` (`prd_agent/signals/external_sentiment_agent.py`) для использования **динамического расчета SL/TP** на основе ATR, адаптированного к текущей волатильности рынка.
